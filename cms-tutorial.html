1 Introduction & What the CMS Controls

The CaltransBizConnect Content Management System (CMS) is a web-based editor built into the platform at /admin-cms.html. It allows authorized administrators to update the text, images, and structured content shown on the public-facing website — without touching any code.

What Can You Edit with the CMS?

CMS ModuleWhat It Controls
PagesHero text, page sections, component content, page metadata (title, description)
FAQ ManagerAll questions and answers on the FAQ page and any FAQ-style sections
Global SettingsSite-wide values: site title, contact email, phone number, social links, logo URL
Media LibraryAll images used in CMS-editable sections; upload, manage, and delete

What the CMS Does NOT Control

Some parts of the website are static or controlled by the platform application and cannot be edited through the CMS:

  • The site navigation menu links
  • User account dashboard pages
  • Form functionality and validation
  • The opportunities listing (live database content)
  • Application processing workflows
  • CSS styling and visual design (requires a developer)
ℹ️
Platform administration is separate

If you need to manage user accounts, approve opportunities, or view analytics, refer to the Platform Admin Guide. The CMS only handles website content.

2 Logging In

How to Access the CMS

  1. Open your browser and navigate to: caltransbizconnect.org/admin-cms.html
  2. The CMS login screen will load. You will see two fields: Email and Password.
  3. Enter the email address associated with your admin account. This is the same email you use to log in to the main platform.
  4. Enter your CMS password. Note: depending on how your account was set up, this may be the same as your platform login password or a separate CMS-specific password.
  5. Click Sign In. If your credentials are correct, the CMS dashboard will load within a few seconds.
⚠️
Session duration

Your CMS session lasts 24 hours. After that, you will be logged out automatically and must sign in again. Always save your work before leaving the CMS, as unsaved changes are lost when your session expires.

If Your Login Fails

  • Double-check for typos in your email address — it must match exactly.
  • Check that Caps Lock is not on when entering your password.
  • If you have recently changed your password via the CMS "Change Password" feature, use the new password.
  • If you still cannot log in, contact the platform administrator to verify your account has CMS access.

How to Log Out

  1. Locate the Logout button in the CMS sidebar (left column, near the bottom).
  2. Click it. Your session token is invalidated immediately and you are returned to the login screen.
  3. Always log out when you are done, especially on shared computers.

4 Pages Management

The Pages module is where you edit the content of all public-facing website pages. Click Pages in the sidebar to open it.

Reading the Page List

The page list displays every page the CMS manages, in display order. Each row shows:

ColumnDescription
Page TitleThe human-readable title shown in the editor
Last UpdatedDate and time of the most recent save
System PageA badge shown for built-in pages that cannot be deleted
ActionsEdit, View (live link), and Delete (custom pages only)

Page Display Order

System pages appear first in this fixed order: Home, For Small Businesses, For Prime Contractors, Opportunities, How It Works, Resources, FAQ, Support Services, Eligibility, Contact. Custom pages you create appear after these, sorted alphabetically.

How to Edit a Page

  1. In the Pages list, find the page you want to edit and click its Edit button.
  2. The Page Editor opens. It has two main areas: Page Metadata at the top and Page Sections below.
  3. Edit the Page Title field if you need to update the title shown in the browser tab and in search results.
  4. Edit the Meta Description field if you need to update the snippet shown in Google search results. Keep it under 160 characters for best results.
  5. To edit a section's content, click the Edit button on that section's row. The Section Editor will open below (or in-line).
  6. Make your changes in the Section Editor fields (see Section 5 for a guide to each field type).
  7. When done editing, click Save at the top of the page editor. Your changes are saved to the database and appear live on the website immediately.
🚫
No auto-save

The CMS does not auto-save. If you navigate away from the page editor without clicking Save, all your changes will be lost. Save frequently, especially after making significant edits.

Editing the Page Header (Hero)

The top of many pages has a "hero" banner with a title, subtitle, and optionally a background image. These are editable in the Page Editor:

FieldWhere It AppearsNotes
Background ImageBehind the hero bannerPick from Media Library; use a wide landscape image (min 1400px wide)
Logo ImageInside the hero, above the titleOptional; used on some branded pages
Logo Alt TextScreen readersAlways required if a logo image is set

How to Reorder Page Sections

Each section in the Page Editor has up-arrow and down-arrow buttons on the left side of its row.

  1. Locate the section you want to move in the section list.
  2. Click the up arrow button to move it one position up, or the down arrow button to move it one position down.
  3. Repeat until the section is in the desired position.
  4. Click Save to preserve the new order.

How to Add a New Section

  1. In the Page Editor, scroll to the bottom of the section list.
  2. Click + Add New Section. A dropdown menu appears listing all available component types.
  3. Select the component type that best fits the content you want to add (see Section 5 for a full reference of component types).
  4. A new empty section of that type is added to the bottom of the list. Click Edit on the new section to fill in its content.
  5. Click Save when finished.

How to Delete a Section

  1. In the Page Editor, locate the section you want to remove.
  2. Click the Delete Section button (trash icon) on that section's row.
  3. A confirmation prompt will appear: "Are you sure you want to delete this section?" Click Delete to confirm.
  4. Click Save to apply. The section is removed permanently from that page.
⚠️
Section deletion is immediate

Once you click Save after deleting a section, the content is gone. There is no Undo. Before deleting, copy the content to a text document if there's any chance you'll need to restore it.

How to Create a New Custom Page

  1. In the Pages list, click + New Page.
  2. Enter a Page Title and a Page Slug. The slug is the URL path — for example, a slug of vendor-resources means the page will be at /vendor-resources.html. Use only lowercase letters, numbers, and hyphens.
  3. Optionally enter a meta description.
  4. Click Create Page. The new page appears in the Pages list.
  5. Click Edit on the new page to start adding sections.
💡
Custom pages are not linked automatically

Creating a page in the CMS does not add it to the site navigation menu. To make a custom page accessible, you must either link to it from another page's content or ask a developer to add it to the nav.

How to View a Live Page

  1. In the Pages list, click the View button (external link icon) next to any page.
  2. The live page opens in a new browser tab, showing exactly what public visitors see.
  3. After saving changes in the editor, refresh this tab to see your updates.

How to Delete a Custom Page

Only custom pages (those without the "System Page" badge) can be deleted. System pages are permanent.

  1. In the Pages list, click Delete on the custom page row.
  2. Confirm in the prompt. The page and all its content are permanently removed from the CMS and will return a 404 error if visited.

5 Section Types Reference

When you add or edit a page section, the editing interface changes depending on the section's component type. Below is a complete reference for every available type.

text Single-line Text

A plain text input for short content like titles, names, or labels. No formatting is allowed. Use this for headings or captions.

textarea Multi-line Text

A larger text area for paragraphs. Basic HTML tags (bold, italic, links) are supported. Use for body copy, descriptions, or summaries.

url URL Field

A validated text input for web addresses. Used for button links, image source URLs, or external references. Always include https://.

select Dropdown

A predefined list of options. Common uses: selecting a layout variant, a color theme, or a content alignment. The available options are fixed by the developer.

checkbox Toggle

A boolean on/off switch. Example: show or hide a section, enable or disable a feature flag, or control visibility of an element.

media Image Picker

Opens the Media Library to select an uploaded image. Set the image and its alt text here. Always provide a meaningful alt text for accessibility compliance.

button CTA Button

Configures a call-to-action button: button text, destination URL, and style (primary/secondary). Used for prominent action links within sections.

card-list Card Grid

A repeating list of cards, each with a title, body text, optional image, and optional button. Use for feature highlights, program overviews, or resource listings. Add, edit, reorder, and delete individual cards.

steps-list Numbered Steps

A numbered process list where each step has a short label and a detail text. Use for how-it-works sequences, onboarding flows, or application processes.

tile-list Tile Grid

A grid of tiles, each with a title, short description, and optional link. Similar to card-list but more compact. Use for quick-link tiles or icon-based navigation sections.

accordion-list Expandable FAQ

A set of collapsible question/answer pairs (accordion). Each item has a question (always visible) and an answer (shown when expanded). Use for inline FAQs or help content within a page.

cta-list CTA Sections

One or more full call-to-action sections, each with a headline, supporting text, and a button. Use for key program actions at the bottom of a page or between major sections.

string-list Simple List

A plain bullet list of text strings with no sub-fields. Use for eligibility requirements, features lists, or quick facts that don't need individual titles.

pathway-card Pathway Card

A specialized card component for "pathway" navigation — used on the Home page to direct users to Small Business or Prime Contractor sections. Has a fixed structure defined by the developer.

Working with List-Type Sections

For any component type with "list" in its name (card-list, steps-list, tile-list, accordion-list, cta-list, string-list), the editing interface provides a list of items that you can manage individually:

ActionHow to Do It
Add a new itemClick the + Add Item button below the item list. A new empty item row is created at the bottom.
Edit an itemClick the Edit pencil icon on the item row. The item's fields expand inline for editing.
Reorder itemsUse the up and down arrow buttons on each item row to change its position in the list.
Delete an itemClick the Delete (trash) icon on the item row. Confirm in the dialog. The item is removed immediately — save the page to finalize.

6 Rich Text Editor

Some fields — particularly long text answers in FAQs, accordion answers, and description fields — display a formatting toolbar above the text area. This is the Rich Text Editor.

Toolbar Buttons

ButtonWhat It DoesWhen to Use
B BoldWraps the selected text in <strong> tags, making it bold on the page.Highlight key terms, requirements, or important phrases. Use sparingly.
I ItalicWraps the selected text in <em> tags, making it italic.Titles of documents, technical terms, or gentle emphasis. Use sparingly.
• ListInserts an unordered (bullet) list <ul><li> structure at the cursor position.Eligibility criteria, document checklists, feature lists.
🔗 LinkInserts an anchor tag <a href=""> at the cursor. You will be prompted for the URL and link text.References to external resources, other pages, or downloadable forms.

Character Count

Fields that have a maximum character limit display a live character counter below the text area (e.g., "143 / 160 characters"). When you exceed the limit, the counter turns red. This most commonly applies to meta description fields.

Tips for Rich Text Fields

  • Select first, then click the toolbar button — highlight the text you want to format before clicking Bold or Italic.
  • Don't paste formatted content from Word or Google Docs — it may carry hidden formatting that breaks the layout. Paste as plain text first (Ctrl+Shift+V on most systems), then format using the toolbar.
  • Test all links before saving — paste the URL into a new browser tab to confirm it works before embedding it in the CMS.
  • Use lists for three or more items — if you have two or fewer items, write them as inline text instead.

7 FAQ Manager

The FAQ Manager controls all questions and answers displayed on the FAQ page and any FAQ accordion sections across the site. Click FAQ Manager in the sidebar to open it.

Viewing and Filtering FAQs

The FAQ Manager shows a table of all FAQ entries. Use these controls to find what you need:

  • Category filter dropdown — show FAQs from one category only. Select "All Categories" to see everything.
  • Search field — filter by keywords in the question text. Results update as you type.

The table displays: Category badge, Question (first 50 characters), Answer snippet (first 100 characters), Status (Active/Inactive), and action buttons.

How to Create a New FAQ

  1. Click + New FAQ above the FAQ table. The FAQ editor modal opens.
  2. Fill in the Category field. Either select an existing category from the dropdown or type a new category name. Categories group FAQs on the FAQ page under headings.
  3. Type the Question in the question textarea. Write it as a complete question (e.g., "How do I submit my capability statement?").
  4. Type the Answer in the answer field. Use the Rich Text Editor toolbar to add formatting if needed. Keep answers concise and to the point.
  5. Set the Status dropdown to Active to make the FAQ visible on the site, or Inactive to save it as a draft.
  6. Click Save FAQ. The new entry appears in the table immediately.

How to Edit an Existing FAQ

  1. Locate the FAQ in the table (use the search or category filter to narrow it down).
  2. Click Edit on that row. The FAQ editor modal opens with the current content pre-filled.
  3. Make your changes to any field: category, question, answer, or status.
  4. Click Save FAQ. The changes take effect on the live FAQ page immediately.

Deactivating vs. Deleting an FAQ

ActionResultWhen to Use
Deactivate (set status to Inactive)FAQ is hidden from the public site but preserved in the databaseTemporarily remove a FAQ without losing the content (e.g., pending legal review, seasonal content)
DeleteFAQ is permanently removed from the databaseThe question is entirely obsolete and will never be needed again

To deactivate: open the FAQ editor, change Status to Inactive, and save.
To delete: click Delete on the FAQ row, then confirm in the dialog.

How to Reorder FAQs with Drag-and-Drop

Within each category, you can reorder FAQs to control the order they appear on the FAQ page.

  1. Filter the FAQ table to show a single category (use the Category filter dropdown).
  2. Click and hold the drag handle (☰ icon) on the left side of any FAQ row.
  3. Drag the row up or down to the desired position, then release.
  4. Repeat for any other rows you want to reorder.
  5. Click Save Order at the top of the table. The new display order is saved to the database.
⚠️
Save Order is a separate save

Reordering FAQs requires clicking the dedicated Save Order button, which is separate from the main Save button. If you close the FAQ Manager without clicking Save Order, your reordering will be lost.

How to Export FAQs

  1. Click the Export FAQs button at the top of the FAQ Manager.
  2. Your browser downloads a JSON file containing all active FAQs, including their categories, questions, answers, and sort order.
  3. This file can be used as a backup or for content review outside the CMS.

When you save Global Settings, the changes take effect on every page of the website immediately. There is no preview mode. Double-check all values before clicking Save.

9 Media Library

The Media Library is a centralized repository for all images used in CMS-editable sections. Click Media Library in the sidebar to open it.

Viewing Uploaded Media

The Media Library displays all uploaded images as a grid of thumbnails. Each image shows:

  • Thumbnail preview — a small preview of the image
  • Filename — the original file name with extension
  • File size — displayed as KB or MB
  • Upload date — when the file was added
  • Alt text — the accessibility description set at upload
  • Download — link to download the original file
  • Delete — button to permanently remove the file

How to Upload an Image

  1. In the Media Library, locate the Upload Image section below the media grid.
  2. Click Choose File (or drag and drop a file onto the upload area). Select your image file from your computer.
  3. Accepted formats: JPEG (.jpg, .jpeg), PNG (.png), GIF (.gif), WebP (.webp), SVG (.svg). Maximum file size: 10 MB.
  4. In the Alt Text field that appears, type a description of the image. This is required by Caltrans accessibility policy. Write what the image shows — for example, "Construction workers installing a guardrail on a California highway" — not generic text like "image" or "photo."
  5. Click Upload. The image is uploaded to the server and appears in the media grid immediately.
  6. You can now select this image from the media picker when editing page sections.
⚠️
Alt text is required — and matters

Alt text is not just a form requirement. Screen reader users cannot see images; they rely entirely on alt text to understand what the image shows. Write descriptive, specific alt text for every image. Do not use filenames as alt text.

Image Size Recommendations

Use CaseRecommended SizeFormat
Hero / Banner background1400 × 600 px minimumJPEG (smaller file size)
Card images600 × 400 pxJPEG or WebP
Logos / Icons200 × 200 px (square)PNG (transparency support) or SVG
Thumbnails400 × 300 pxJPEG or WebP
💡
Optimize images before uploading

Large image files slow down page load times for all visitors. Before uploading, compress your images using a tool like Squoosh or TinyPNG. Aim for under 300 KB for most images.

How to Select an Image When Editing a Page Section

  1. While editing a page section that has a media-type field, click the Choose Image or Select from Library button.
  2. The Media Picker modal opens, showing all uploaded images. Scroll to find the image you want.
  3. Click on the image thumbnail to select it. The modal closes automatically and the image is applied to the section.
  4. The image's existing alt text is filled in automatically, but you can override it for this specific usage if the context requires a different description.
  5. Click Save to preserve the image selection.

How to Delete a Media File

  1. In the Media Library grid, locate the image you want to delete.
  2. Click the Delete button below that image's thumbnail.
  3. Confirm in the dialog. The file is permanently deleted from the server.
🚫
Check before deleting

If an image is currently used in a page section, deleting it from the Media Library will cause that section to display a broken image placeholder. Before deleting, search page sections for any use of that image and replace it with another image first.

10 Changing Your Password

You can update your CMS login password directly within the CMS — no need to contact a developer or administrator.

How to Change Your CMS Password

  1. Click Change Password in the CMS sidebar (near the bottom).
  2. In the form that appears, enter your Current Password in the first field.
  3. Enter your new password in the New Password field. Requirements:
    • Minimum 8 characters
    • Mix of uppercase and lowercase letters recommended
    • Include at least one number or special character
  4. Enter your new password again in the Confirm New Password field to ensure there are no typos.
  5. Click Update Password. A confirmation message will appear. Your new password takes effect immediately.
  6. The next time you log in to the CMS, use the new password.
⚠️
Password change logs you out of other sessions

Changing your password invalidates any existing CMS sessions. If you are logged in on multiple devices, you will need to log in again with the new password on each device.

Password Best Practices

  • Change your password every 90 days, or immediately if you suspect it has been compromised.
  • Never use a password you use on any other website or application.
  • Do not share your CMS password with colleagues. Each admin should have their own account.
  • Consider using a password manager (like 1Password or Bitwarden) to generate and store a strong, unique password.

11 Best Practices

Content Quality Standards

  • Write in plain language. Avoid jargon. If a 10th-grader can't understand your content, rewrite it. Caltrans accessibility standards require plain language across all public-facing content.
  • Be consistent. Use the same terminology throughout the site. For example, always write "Small Business Enterprise" not "Small Business" on first reference, followed by "(SBE)" in parentheses.
  • Keep sentences short. Aim for 20 words or fewer per sentence in body copy.
  • Proofread before saving. Changes go live immediately. Read your content aloud before clicking Save.

Accessibility Requirements

  • Always provide alt text for images. Never leave the alt text field blank when uploading to the Media Library.
  • Don't use color alone to convey meaning. If you use formatting in rich text, make sure the meaning is also clear from the text itself.
  • Use heading hierarchy correctly. In FAQ answers and section body text, use bold or strong for emphasis — do not skip to heading styles for stylistic purposes.
  • Test your links. Every URL you add must be valid and working at the time you publish it.

Preview Before Publishing

  1. After saving, click the View button (live page link) on the page in the Pages list.
  2. Review the page in the live view on a desktop-sized window.
  3. Resize your browser or use browser DevTools to check the mobile view.
  4. If anything looks wrong, return to the CMS and make corrections.

Content Review Workflow

For significant content updates (especially to program eligibility requirements, fee information, or legal language), follow this review process before publishing:

  1. Draft the content in a shared Google Doc or Word document.
  2. Have at least one colleague review the draft for accuracy and clarity.
  3. Obtain approval from the program manager or legal team if the content involves program rules or compliance language.
  4. Only then enter the reviewed, approved content into the CMS and save.
✅
Good content update cadence

Plan to review and refresh the FAQ and program overview pages at least quarterly. Outdated information — especially around deadlines, contact details, and program requirements — erodes user trust and can cause compliance issues.

12 Troubleshooting

Common Issues and Resolutions

My login is failing even though my password is correct

  • Confirm you are going to /admin-cms.html (not /login.html, which is the platform login for vendors and agencies).
  • Check for browser autofill filling in an old password — clear the field and type manually.
  • Your session may have been invalidated. Try logging in from a fresh browser tab.
  • If the problem persists, contact the platform administrator to reset your CMS credentials.

I saved a page but my changes are not showing on the live site

  • Hard-refresh the live page: press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) to bypass the browser cache.
  • Open the live page in a private/incognito window — this always loads fresh content.
  • If the old content still shows after a hard refresh, check that the save action completed successfully (look for the green "Saved" confirmation in the CMS).

The Pages list is empty or shows an error

  • This is usually a temporary server issue. Wait 30 seconds and refresh the CMS page.
  • If the error persists, the server may be down. Contact the technical administrator.

I can't upload an image — the upload button is greyed out or gives an error

  • Verify the file is one of the accepted formats: JPEG, PNG, GIF, WebP, SVG.
  • Check the file size — it must be under 10 MB. Compress the image and try again.
  • Ensure you have filled in the Alt Text field — the upload will not proceed without it.
  • If the error message says "Upload failed" after a valid file and alt text, the server storage may be full — contact the technical administrator.

My FAQ reordering was not saved

  • After dragging FAQs to their new positions, you must click the Save Order button (not just any Save button). This is a dedicated action.
  • If Save Order is not visible, scroll to the top of the FAQ table.

I deleted a section or page by mistake

Deletions in the CMS are permanent and cannot be undone from within the tool. If the deletion was very recent, contact the server administrator immediately to request a database restore from backup. For small sections, recreate the content manually using the Add Section feature.
