1 Introduction & Access

About This Guide

This guide is for Caltrans staff who have been granted admin or caltrans_admin platform roles on CaltransBizConnect. It covers every administrative action available, including user account management, opportunity review and approval, application oversight, and platform analytics.

ℹ️
Who this guide is for

Platform admins manage accounts and operations. If you manage website content (page text, FAQs, images), refer to the separate CMS Admin Guide.

Your Admin Pages

As a platform admin, you have access to three dedicated pages:

PageURLPurpose
Analytics/admin-analytics.htmlView real-time platform metrics and user activity
User Management/admin-users.htmlCreate, edit, approve, suspend, and delete user accounts
CMS (Content)/admin-cms.htmlEdit page content, FAQs, and site settings (see CMS Guide)

How to Log In

  1. Navigate to caltransbizconnect.org/login.html in your browser.
  2. Enter your admin email address and password, then click Log In.
  3. If your credentials are valid and your account has admin privileges, you will be redirected automatically to the admin analytics dashboard.
  4. Use the navigation links in your dashboard header to move between analytics, user management, and other admin areas.
⚠️
Session expiry

Admin sessions expire after 7 days of inactivity. If you see a "Please log in" message, your session has expired — return to the login page and sign in again.

Role Differences: admin vs. caltrans_admin

Both roles have identical access to the admin UI. The caltrans_admin role is reserved for Caltrans Civil Rights staff and is the highest-privilege role on the platform. The admin role is used for delegated administrators. Both can perform all actions described in this guide.

2 Analytics Dashboard

The Analytics Dashboard (/admin-analytics.html) gives you a real-time view of platform health. It is the first page you see after logging in as an admin.

Top-Line Metrics

Four key figures are displayed at the top of the dashboard:

MetricWhat It CountsWhen to Act
Total Registered UsersAll accounts regardless of statusRapid drops may indicate account deletions
Total OpportunitiesAll opportunities regardless of statusMonitor against expected agency posting activity
Active OpportunitiesOpportunities with status openLow count = few open bids; follow up with agencies
New Users This WeekAccounts registered in the past 7 daysHigh spikes may indicate automated signups

Users by Type Breakdown

A visual bar chart shows the distribution of account types across the platform:

  • Small Business — certified SBE/DBE firms registered to apply for subcontracting work
  • Prime Contractor — agencies and prime contractors posting opportunities
  • Admin — internal staff accounts

The percentage shown is each type as a share of all registered users. A healthy ratio typically shows small businesses as the majority.

Opportunities by Status

This breakdown shows the pipeline health of all posted opportunities:

StatusMeaning
openPublished and accepting applications from small businesses
pendingPosted by an agency but not yet reviewed/approved by admin
closedPast due date or manually closed; no new applications accepted
awardedContract has been awarded to a small business
💡
Watch the pending count

A growing "Pending" count means agencies are posting opportunities that haven't been reviewed yet. You or another admin should approve or reject these in the Opportunity Manager (see Section 4).

Recent Registrations Table

The bottom of the analytics page shows the 10 most recently registered users, including their name/business, account type badge, and join date. Each row includes a Manage link that opens that user's record directly in the User Management page.

Refreshing Data

  1. Click the Refresh button at the top of the analytics page.
  2. All four metrics and both breakdowns will reload from the live database. The recent registrations table also refreshes.

Data does not auto-refresh. Reload the page or click the Refresh button to see current figures.

3 User Management

User Management (/admin-users.html) is your primary tool for managing who has access to the platform. You can view, create, edit, approve, suspend, and delete user accounts from this page.

Reading the User Table

The main table lists every registered account. Each row shows:

ColumnDescription
User / BusinessBusiness or organization name with the contact email below it
Account TypeColored badge: Small Business, Prime Contractor, or Admin
StatusCurrent account state (see status reference below)
Join DateWhen the account was created
ActionsButtons to manage the account (context-dependent)

Account Status Reference

StatusMeaningCan Log In?
PendingAccount created but not yet reviewed by adminLimited access
ActiveFully active, normal accessYes
VerifiedActive + SBE/DBE certification confirmedYes
SuspendedAccount temporarily disabled by adminNo

Searching and Filtering

  1. Locate the search bar at the top of the user table labeled "Search users."
  2. Type any part of the business name, contact name, or email address. The table filters in real time — no button to press.
  3. Clear the search field to restore all users.

How to Approve a New User

When a new user registers, their account status is Pending. They have limited platform access until an admin approves them.

  1. In the user table, locate the pending account. Filter by status or search by business name to find it quickly.
  2. Click the Approve button in that user's Actions column. This button only appears for accounts with Pending status.
  3. The status badge updates immediately to Active and the Approve button disappears. The user can now log in with full access.
  4. If the business is a certified SBE/DBE, open their record (click Manage) and change their status to Verified Small Business using the Status dropdown, then click Save Changes.
💡
Verify certification numbers

Before setting a user to Verified status, confirm their SBE certification number in CalTrans CCOP or the relevant certifying agency database.

How to Suspend or Deactivate a User

Use suspension for policy violations, account security concerns, or temporary holds. Suspension prevents login but preserves the account and all its data.

  1. Locate the user's row in the table.
  2. Click Deactivate in the Actions column. This button only appears for accounts that are currently Active or Verified.
  3. Confirm the action in the dialog that appears. The status updates to Suspended immediately.
  4. To re-activate, click Manage, change the Status dropdown back to Active or Verified Small Business, and click Save Changes.
🚫
Suspension vs. Deletion

Suspension is reversible. Deletion is permanent and removes the user's account, all their saved opportunities, and any applications they submitted. Only delete accounts when you are certain they will never be needed again.

How to Create a New User Account

Admins can manually create accounts — for example, when onboarding an agency that needs platform access before their self-registration is processed.

  1. Click the + Add New User button above the user table.
  2. A modal dialog titled "Create User Account" will open. Fill in all required fields:
FieldRequired?Notes
User / Organization NameYesBusiness or organization name (not a personal name)
Email AddressYesMust be unique — this is the login email
PasswordYes (new users)Minimum 8 characters. Share securely with the user; they should change it after first login
Account TypeYesSmall Business, Prime Contractor, or Administrator
Account StatusYesSet to Active for immediate access or Pending if review is needed
  1. Click Create User. The new account appears in the table immediately.
  2. Communicate the login credentials to the new user through a secure channel (not email in plaintext).

How to Edit an Existing User

  1. Find the user in the table and click Manage in their Actions column.
  2. The "Manage User Account" modal opens, pre-filled with their current information.
  3. Update any of the following fields as needed:
    • User / Organization Name — correct typos or update after a name change
    • Email Address — update if the user's contact email changed
    • Password — leave blank to keep the current password; enter a new value to reset it
    • Account Type — change role if the user's relationship to the platform changes
    • Account Status — move between Active, Pending, Suspended, and Verified Small Business
  4. Click Save Changes to apply. Changes take effect immediately.
⚠️
Changing account type

Changing a user from Small Business to Administrator or Prime Contractor will change their dashboard and what pages they can access. Confirm the change with your supervisor before upgrading any account to Admin or caltrans_admin.

How to Delete a User

  1. Click Manage for the user you wish to delete.
  2. In the modal, scroll to the bottom and click the Delete User button.
  3. A confirmation prompt will appear: "Are you sure you want to delete this account? This action cannot be undone." Click Confirm Delete only if you are certain.
  4. The account is permanently removed from the database. The modal closes and the user disappears from the table.
🚫
Deletion is irreversible

Deleted users cannot be recovered. Their capability statements, saved opportunities, messages, and application history are permanently erased. Prefer Suspension over Deletion in all but the most clear-cut cases.

4 Opportunity Management

Agencies and prime contractors post subcontracting opportunities on the platform. Before an opportunity appears to small businesses, a platform admin must review and approve it. This section covers the full approval workflow and ongoing opportunity management.

The Opportunity Approval Workflow

When an agency posts a new opportunity, it enters the system with a status of pending. It is invisible to small businesses until an admin changes its status to open. The flow is:

Agency posts → pending → Admin reviews → open (published) → Bids close → closed → Award selected → awarded

How to Review and Approve a Pending Opportunity

  1. From your admin dashboard, look for the Pending Reviews count on the analytics page, or navigate directly to the opportunities section in the admin dashboard.
  2. Locate opportunities with Pending status. Review the following before approving:
    • Title and scope summary — is the work described clearly and professionally?
    • Due date — is it at least 5 business days in the future?
    • Estimated value — is a dollar amount provided?
    • NAICS codes — is at least one code listed so the matching system works?
    • District — is a Caltrans district or "Statewide" selected?
    • Certifications — are SBE/DBE requirements accurately stated?
  3. If the opportunity meets standards, click Approve. The status changes to open and the opportunity is immediately visible to small businesses on the Opportunities page.
  4. If the opportunity needs corrections, contact the posting agency directly and ask them to update the posting before you approve it.
💡
Pending count on analytics

Check the Analytics Dashboard regularly. The "Pending Reviews" metric shows how many opportunities are waiting for your approval. A target of 0 pending items means small businesses have the most up-to-date opportunities available.

How to Edit an Opportunity

Admins may edit any opportunity, including those posted by agencies. This is typically used to fix factual errors, update due dates, or add missing NAICS codes.

  1. Navigate to the opportunity in the admin interface and click Edit.
  2. The opportunity editor opens with all current values pre-filled. Modify only the fields that need correction.
  3. Click Save. Changes are applied immediately, even for published opportunities.

Editable Opportunity Fields

FieldNotes

How to Close an Opportunity

When the due date passes, opportunities should be closed to prevent further applications.

  1. Open the opportunity editor.
  2. Change the Status field to closed.
  3. Save. The opportunity will no longer appear as active on the Opportunities page but remains searchable in its history.

How to Mark an Opportunity as Awarded

  1. Open the opportunity editor for a closed opportunity.
  2. Change the Status to awarded.
  3. Optionally add the winning small business name in the tags field for reference.
  4. Save. The opportunity displays an "Awarded" badge on the platform.

How to Delete an Opportunity

  1. Open the opportunity in the admin interface.
  2. Click Delete Opportunity.
  3. Confirm the deletion. The opportunity and all associated applications are permanently removed.
🚫
Prefer closing over deleting

Deleting an opportunity removes all application records associated with it, which may affect small businesses that applied and agency reporting. In most cases, set the status to closed rather than deleting.

5 Application Review

When a small business applies to an opportunity, an application record is created. Normally, the posting agency reviews and acts on applications. However, admins can view all applications across the platform and override statuses when needed.

Application Statuses Explained

StatusMeaningWho Sets It
pendingApplication submitted, not yet reviewedSystem (on submission)
under_reviewAgency is actively reviewing the applicationAgency / Admin
approvedApplication approved for the opportunityAgency / Admin
declinedApplication rejectedAgency / Admin
awardedThis applicant was awarded the contractAgency / Admin

Viewing Applications

  1. Navigate to the Applications section in the admin dashboard.
  2. Use the filter controls to narrow by opportunity, small business, or status.
  3. Click any application row to view the full details: applicant profile, capability statement, applied date, and status history.

Updating Application Status (Admin Override)

  1. Open the application record.
  2. Use the Status dropdown to change to the appropriate status.
  3. Click Save. The change is logged and the applicant's dashboard will reflect the new status.
⚠️
Communicate status changes to applicants

Status changes trigger no automatic email notification to the small business. If you change an application status as an admin override, consider sending the small business a direct message through the platform or an email to inform them.

Viewing Applications for a Specific Opportunity

  1. Open the opportunity in the admin interface.
  2. Click the View Applications button.
  3. All applications for that opportunity are listed with their current status, applicant name, and submission date.

Viewing All Applications from a Specific Small Business

  1. Open the small business's account in User Management (click Manage).
  2. From the user profile, click View Applications to see every opportunity this business has applied to.

6 Security Best Practices

Protecting Your Admin Account

  • Use a strong, unique password — minimum 12 characters, mix of letters, numbers, and symbols. Never reuse a password from another account.
  • Never share your admin credentials — if another staff member needs admin access, create them a separate account.
  • Log out when done — especially on shared or government-issued computers. Your session lasts up to 7 days if not explicitly logged out.
  • Access on secured networks only — avoid logging into the admin panel on public Wi-Fi. Use your agency VPN if working remotely.

Account Lifecycle Hygiene

  • Conduct a quarterly audit of all admin accounts. Suspend or delete accounts for staff who have left the program.
  • Review the Recent Activity feed monthly for unexpected registrations or changes.
  • Verify that no user has an inappropriately elevated account type (e.g., a contractor account set to Admin).

Suspicious Activity

If you observe any of the following, contact the technical team immediately:

  • Unexpected admin account creations
  • Opportunities approved that you did not approve
  • Unusual spikes in new user registrations
  • User accounts with unknown email domains
  • Any content change you cannot explain
🚨
If you suspect a compromise

Immediately change your admin password and notify the Caltrans Civil Rights IT contact. Do not attempt to investigate on your own — preserve the audit trail for forensic analysis.

7 Troubleshooting

Common Issues and Resolutions

I can't log in to the admin panel

  • Verify you are using your admin account email, not a regular user account.
  • Confirm your password hasn't expired or been changed by another admin.
  • Clear your browser cache and cookies, then try again.
  • If still unable to log in, contact the technical administrator to verify your account status and role in the database.

The user table shows no data

  • Check that the search field is empty — an active search filter can hide all users if no matches are found.
  • Click the Refresh button or reload the page.
  • If the table consistently shows no data, the database connection may be down — contact the server administrator.

An opportunity I approved is not showing on the public Opportunities page

  • Confirm the opportunity status is open, not pending or closed.
  • Check the due date — if the due date has already passed, the opportunity may be automatically hidden.
  • Try viewing the page in a private/incognito browser window to rule out a caching issue.

Analytics data looks incorrect

  • Click the Refresh button to force a live reload from the database.
  • Analytics figures are not cached; a discrepancy may indicate a database query issue. Report persistent inaccuracies to the technical team.

I accidentally deleted a user account

Account deletions are permanent and cannot be undone from the admin UI. If the deletion was recent and the database has a backup, contact the server administrator immediately to request a restore. Otherwise, you will need to create a new account for the affected user and inform them to re-register their profile information.
