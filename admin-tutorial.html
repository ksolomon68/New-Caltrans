Introduction & Access

About This Guide

This guide is for Caltrans staff who have been granted admin or caltrans_admin roles on CaltransBizConnect. It covers every administrative action available: user account management, opportunity review and approval, application oversight, and platform analytics.

ℹ️
Who this guide is for

Platform admins manage accounts and operations. If you manage website content (page text, FAQs, images), see the CMS Admin Guide.

Your Admin Pages

PageURLPurpose
Analytics/admin-analytics.htmlView real-time platform metrics and user activity
User Management/admin-users.htmlCreate, edit, approve, suspend, and delete user accounts
CMS (Content)/admin-cms.htmlEdit page content, FAQs, and site settings

How to Log In

1
Navigate to caltransbizconnect.org/login.html in your browser.
2
Enter your admin email address and password, then click Log In.
3
If your credentials are valid and your account has admin privileges, you will be redirected automatically to the admin analytics dashboard.
4
Use the navigation links in your dashboard header to move between analytics, user management, and other admin areas.
⚠️
Session expiry

Admin sessions expire after 7 days of inactivity. If you see a "Please log in" message, your session has expired — return to the login page and sign in again.

Role Differences: admin vs. caltrans_admin

Both roles have identical access to the admin UI. The caltrans_admin role is reserved for Caltrans Civil Rights staff and is the highest-privilege role on the platform. The admin role is used for delegated administrators. Both can perform all actions in this guide.

The Analytics Dashboard (/admin-analytics.html) gives you a real-time view of platform health. It is the first page you see after logging in as an admin.

User Management

User Management (/admin-users.html) is your primary tool for managing who has access to the platform. You can view, create, edit, approve, suspend, and delete user accounts from this page.

Reading the User Table

ColumnDescription
User / BusinessBusiness or organization name with the contact email below it
Account TypeColored badge: Small Business, Prime Contractor, or Admin
StatusCurrent account state (see reference below)
Join DateWhen the account was created
ActionsContext-dependent action buttons

Account Status Reference

StatusMeaningCan Log In?
PendingAccount created but not yet reviewed by adminLimited access
ActiveFully active, normal accessYes
SuspendedAccount temporarily disabled by adminNo

Searching and Filtering

Use the search bar at the top of the user table to filter in real time by business name, contact name, or email address. Clear the field to restore all users.

How to Approve a New User

When a new user registers, their status is Pending. They have limited platform access until an admin approves them.

1
In the user table, locate the pending account. Use the search field to find it quickly.
2
Click the Approve button in that user's Actions column. This button only appears for accounts with Pending status.
3
The status badge updates immediately to Active. The user can now log in with full access.

How to Suspend or Deactivate a User

Suspension prevents login but preserves the account and all its data. Use it for policy violations, security concerns, or temporary holds.

1
Locate the user's row in the table.
2
Click Deactivate in the Actions column. This button only appears for Active accounts.
3
Confirm the action. The status updates to Suspended immediately.
4
To re-activate: click Manage, change the Status dropdown back to Active or Verified, and click Save Changes.
🚫
Suspension vs. Deletion

Suspension is reversible. Deletion is permanent and removes the user's account, saved opportunities, and all applications they submitted. Only delete accounts when you are certain they will never be needed again.

How to Create a New User Account

1
Click + Add New User above the user table.
2
Fill in all required fields in the modal:
  • User / Organization Name — business or organization name
  • Email Address — must be unique; this is the login email
  • Password — minimum 8 characters; share securely with the user
  • Account Type — Small Business, Prime Contractor, or Administrator
  • Account Status — Active for immediate access, Pending if review is needed
3
Click Create User. The new account appears in the table immediately.
4
Communicate the login credentials to the new user through a secure channel — not email in plaintext.

How to Edit an Existing User

1
Click Manage on the user's row to open the "Manage User Account" modal.
2
Update any fields as needed:
  • User / Organization Name — correct typos or update after a name change
  • Email Address — update if the user's contact email has changed
  • Password — leave blank to keep the current password; enter a new value to reset it
  • Account Type — change role if the user's relationship to the platform changes
  • Account Status — move between Active, Pending, Suspended, and Verified
3
Click Save Changes. Changes take effect immediately.
⚠️
Changing account type

Changing a user from Small Business to Administrator changes their dashboard and page access. Confirm with your supervisor before upgrading any account to Admin or caltrans_admin.

How to Delete a User

1
Click Manage for the user you wish to delete.
2
In the modal, click the Delete User button at the bottom.
3
Confirm the dialog. The account is permanently removed and the user disappears from the table.
🚫
Deletion is irreversible

Deleted users cannot be recovered. Their capability statements, saved opportunities, messages, and application history are permanently erased. Prefer Suspension over Deletion in all but the clearest cases.

Opportunity Management

When an agency posts a subcontracting opportunity, it enters the system as pending. It is invisible to small businesses until an admin approves it. This section covers the full approval workflow and ongoing management.

The Opportunity Workflow

Agency posts → pending → Admin reviews → open (published) → Bids close → closed → Award selected → awarded

How to Review and Approve a Pending Opportunity

1
From your admin dashboard, check the Pending Reviews count on the analytics page or navigate to the Opportunity Manager section.
2
Locate opportunities with Pending status. Review the following before approving:
  • Title and scope summary — clearly and professionally described?
  • Due date — at least 5 business days in the future?
  • Estimated value — a dollar amount provided?
  • NAICS codes — at least one code listed?
  • District — a Caltrans district number or "Statewide" selected?
  • Certifications — SBE/DBE requirements accurately stated?
3
If the opportunity meets standards, click Approve. The status changes to open and the opportunity is immediately visible to small businesses.
4
If corrections are needed, contact the posting agency and ask them to update the posting before you approve it.

How to Edit an Opportunity

1
Navigate to the opportunity in the admin interface and click Edit.
2
The opportunity editor opens with all current values pre-filled. Modify only the fields that need correction.
3
Click Save. Changes apply immediately, even for published opportunities.

Editable Opportunity Fields

FieldNotes
TitleShort, clear description of the work being subcontracted
Scope SummaryFull description of scope, requirements, and deliverables
DistrictCaltrans district number (1–12) or Statewide
CategoryWork category (e.g., Construction, Environmental, IT)
Estimated ValueDollar amount; required for small businesses to assess fit
Due Date & TimeSubmission deadline

How to Close an Opportunity

1
Open the opportunity editor.
2
Change the Status field to closed.
3
Save. The opportunity no longer appears as active on the Opportunities page but remains in its history.

How to Mark an Opportunity as Awarded

1
Open the opportunity editor for a closed opportunity.
2
Change the Status to awarded. Optionally note the winning small business in the tags field.
3
Save. The opportunity displays an "Awarded" badge on the platform.
🚫
Prefer closing over deleting

Deleting an opportunity permanently removes all associated application records, which affects small businesses who applied and agency reporting. Set the status to closed rather than deleting in almost all cases.

Application Review

When a small business applies to an opportunity, an application record is created. The posting agency normally reviews applications, but admins can view all applications across the platform and override statuses when needed.

Application Statuses

StatusMeaningWho Sets It
pendingApplication submitted, not yet reviewedSystem (on submission)
under_reviewAgency is actively reviewing the applicationAgency / Admin
approvedApplication approved for the opportunityAgency / Admin
declinedApplication rejectedAgency / Admin
awardedThis applicant was awarded the contractAgency / Admin

Updating Application Status (Admin Override)

1
Navigate to the Applications section and open the application record.
2
Use the Status dropdown to change to the appropriate status.
3
Click Save. The change is logged and the applicant's dashboard reflects the new status.
⚠️
Communicate status changes to applicants

Status changes do not trigger automatic email notifications to the small business. If you change a status as an admin override, send the small business a direct message or email to inform them.

Viewing Applications for a Specific Opportunity

1
Open the opportunity in the admin interface.
2
Click the View Applications button to see all applications for that opportunity, with status, applicant name, and submission date.

Viewing All Applications from a Specific Small Business

1
Open the small business's account in User Management (click Manage).
2
From the user profile, click View Applications to see every opportunity this business has applied to.

Security Best Practices

Protecting Your Admin Account

  • Use a strong, unique password — minimum 12 characters, mix of letters, numbers, and symbols. Never reuse a password from another account.
  • Never share your admin credentials — if another staff member needs admin access, create them a separate account.
  • Log out when done — especially on shared computers. Your session lasts up to 7 days if not explicitly logged out.
  • Access on secured networks only — avoid logging in on public Wi-Fi. Use your agency VPN when working remotely.

Account Lifecycle Hygiene

  • Conduct a quarterly audit of all admin accounts. Suspend or delete accounts for staff who have left the program.
  • Review the Recent Activity feed monthly for unexpected registrations or changes.
  • Verify that no user has an inappropriately elevated account type (e.g., a contractor set to Admin).

Credential Rotation Schedule

CredentialRecommended Rotation
Admin account passwordEvery 90 days
CMS admin passwordEvery 90 days
Email service app passwordEvery 6 months or upon any personnel change
JWT secret (server config)Upon any suspected compromise; otherwise annually
🚨
If you suspect a compromise

Immediately change your admin password and notify the Caltrans Civil Rights IT contact. Do not attempt to investigate on your own — preserve the audit trail for forensic analysis.

Troubleshooting

I can't log in to the admin panel

  • Verify you are using your admin account email, not a regular user account.
  • Check that your password hasn't been changed by another admin.
  • Clear your browser cache and cookies, then try again.
  • If still unable, contact the technical administrator to verify your account status and role.

The user table shows no data

  • Check that the search field is empty — an active filter can hide all users if no matches are found.
  • Click Refresh or reload the page.
  • If the table consistently shows no data, the database connection may be down — contact the server administrator.

An approved opportunity is not showing on the public Opportunities page

  • Confirm the opportunity status is open, not pending or closed.
  • Check the due date — if it has already passed, the opportunity may be automatically hidden.
  • Try viewing the page in a private/incognito browser window to rule out a caching issue.

I accidentally deleted a user account

Account deletions are permanent and cannot be undone from the admin UI. If the deletion was very recent, contact the server administrator to request a database restore from backup. Otherwise, create a new account for the affected user and inform them to re-enter their profile information.
